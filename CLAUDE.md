# CLAUDE.md — Into Place Engineering Conventions

You are the primary development agent for this project. The developer is a solo builder, and the timeframe is a 72-hour hackathon. **Speed and demo-ability take priority over engineering perfection.**

## The project in one sentence

Real place archives → user curation → AI-generated / manual collage storyboard frames → fal image-to-video → FFmpeg assembly → short place-history film. See README.md for details; `spec/` is the sole authority for implementation details.

## Tech stack and structure

- Next.js App Router + TypeScript + Tailwind, deployed on Vercel.
- No database: project state is read from/written to `data/project.json`; assets and outputs are written to disk per the SPEC §5 directory conventions.
- All fal calls happen in a server route; `FAL_KEY` only exists in `.env.local` / Vercel env, and **must never appear in client-side code, logs, git, or chat content**.
- Video jobs always use queue mode (submit + poll); never wait synchronously.

## fal usage discipline (strictly enforced)

1. Before calling any model, first look up that endpoint's **schema and same-day pricing** via fal MCP; the model names in SPEC §4 are candidates, not facts.
2. **Budget has been relaxed (developer decision 2026-07-18, quality first)**: normal calls do not need per-call confirmation; **ask the developer first when a single estimated cost is > $5 or when cumulative spend inflates abnormally**.
3. Choose parameters by target quality (16:9 unchanged); no longer force the 5s/720p test tier; model selection is quality-first, with cheap candidates only as a control/comparison.
4. After 3 generation attempts per shot (`generation.attempts`), stop to analyze and suggest revising the collage or prompt rather than continuing to re-roll — this is anti-derailment discipline, not cost-saving discipline, and is retained.
5. Write each generation's request_id, model, parameters, and cost into the shot JSON to keep it traceable.

## Prompt rules

- Do not hand-write natural-language prompts: everything is compiled from structured shot JSON by `lib/prompt-compiler.ts`.
- Fixed block of negative/protective constraints:
  `no face changes, preserve printed text` + `Diegetic sounds only. No music. No dialogue. No subtitles.`

## Authenticity red lines (product principles, and equally code rules)

- Generated outputs can never be labeled/presented as archives; generated shots must carry an annotation in the UI;
- The source / license field of every asset is mandatory.

## Asset preprocessing v2 (strictly enforced)

1. Preprocessing must be driven by a declarative recipe. First locate the asset content and clearly write out the objects to keep/exclude, then set the normalized crop after EXIF straightening; irrelevant objects such as grayscale bars, rulers, and scan backing plates must not enter the final output. `asset_013` is a PDF awaiting manual page selection — keep skipping it.
2. fal only generates the subject mask, and the request must explicitly set `apply_mask:false`; the original RGB must not be handed to the model for repainting. Color grading, cropping, alpha compositing, edges, white borders, and shadows are all done locally and deterministically.
3. Super-resolution and any resize upscaling are forbidden. For low-resolution assets like 001, 008, 012, only write a quality warning and limit the scene scale.
4. tone is `source | mono | sepia`; edge is `scissor | torn | none`. Historical assets default to mono, modern photos to source; paper cards default to torn with a stable seed, transparent subjects to scissor, backgrounds to none. In the future the UI will allow global selection and per-asset overrides, but the recipe defaults remain the fallback source of truth.
5. Output names must carry a role suffix: `*_card.png`, `*_cutout.png`, `*_bg.png`; scenes and the manifest may only reference the canonical names.
6. fal mask allows at most two prompt/ROI attempts. The fallback order is fixed as SAM → local silueta/rembg → cropped paper card; bad masks must not be published silently.
7. The cache validates source hash, recipe hash, tool version, and mask configuration together. `--force` only re-runs the local pixel stage; only `--refresh-mask` can incur a charge again. Provenance must record pixel source, operation chain, request_id, model, parameters, price-check time, cost, hash, warnings, and review status, and must never record secrets.
8. All results are first written to staging, and are atomically published only after passing alpha/coverage/bbox/connected-region/dimension/RGB-source checks and manual contact sheet review.
9. The recipe's source review and visual review must be recorded separately. `review.visual: rejected` must set both `publish:false` and an explicit `fallbackRecipeId`; that output must not enter `assets/cutouts`, the manifest, or scenes. Changing review/publish status must not invalidate an already-paid-for mask cache.


## Known asset-acquisition limitations (continuously updated, sync after completing a task)

- **NYPL Digital Collections**: `digitalcollections.nypl.org` has Incapsula bot blocking, so a direct curl will 403; you must first use WebFetch to read the item page to get the internal numeric image id, then download via `images.nypl.org/index.php?id=...&t=X`. `t=w` is just a preview tier (~760px long edge); **you can obtain much larger tiers without an API key**: stereograph/print-type items have a JPEG tier `t=g` (about 6700–8900px long edge); other items go up to `t=u` (TIFF master, about 2600–3300px long edge, needs transcoding). Available tiers vary by item type; for new items, probe `t=` values in alphabetical order and verify the actual pixel dimensions (do not rely on file byte size alone). See the known-issues table in `data/day0-ri-archive-notes.md`.
- **Wikimedia Commons**: extremely large original files (especially HABS `.tif` files) get rate-limited with a prompt to use the thumbnail instead; the `Special:FilePath` redirect + a width-specified thumb URL is a stable fallback, and a derivative at the 3840px long-edge level is entirely sufficient as a collage layer.
- **Scanned-PDF-type archive assets** (such as asset_013 *Ten Days in a Mad-House*) cannot be used directly as image layers; a human must first provide the target page number, then do a single-page extraction — see the process in `data/day0-ri-archive-notes.md`.
- **Downloading rembg models on a cloud container**: the network policy only allows package-manager sources; direct model links from GitHub releases / Hugging Face / third-party CDNs all 403. Workable solution: npm `@rmbg/model-u2netp` and `@rmbg/model-silueta` pack rembg's official weights in chunks into npm packages; after concatenating them in order, the md5 matches rembg's expected value, and placing them in `~/.u2net/` makes them usable offline (`REMBG_MODEL=silueta`; concatenation command in `scripts/README.md`). Cutout quality: usable for photos/halftone print, not usable for line engravings (those only go the paper-card route).

## Storyboard-to-video pipeline

`scripts/cutout.mjs` (cutout/rough edge/drop shadow) → Storyboard generated frame or manual collage → reviewed 16:9 start frame →
`lib/prompt-compiler.ts` compiles the motion prompt → fal I2V queue → shot-by-shot review → FFmpeg concatenation/transitions/mixing.

- 2026-07-19 architecture decision: remove the old browser-recording/layered-parallax CLI pipeline; do not reintroduce a headless browser or a standalone video-rendering framework. Manual collage static export prefers the browser Canvas; use the existing Sharp when server-side compositing is needed.
- Both generated frames and manual collage frames must record the corresponding beat, referenced asset IDs, prompt/edit history, and generation provenance; only reviewed frames may enter I2V.
- On manual collage export, only reviewed `assets/cutouts/*.png` and user handwriting may be combined; a generated flat frame must not be handed back to a segmentation model to reverse-find layers.
- When sharp cuts out a shape via a mask, you must use `joinChannel(grayscale mask)` to attach the mask's grayscale values directly as the alpha; `composite({blend:"dest-in"})` recognizes the input image's **own** alpha channel, so feeding it a plain grayscale mask with no alpha is treated as "opaque everywhere" and cuts out nothing (been there — the output is a solid rectangle instead of the mask shape).
- Local environment dependencies: FFmpeg (winget `Gyan.FFmpeg`) and rembg (`pip install rembg[cli] onnxruntime`, executables in `%APPDATA%\Roaming\Python\Python314\Scripts`) are both installed, and the executable directories are already permanently added to the User PATH — a newly opened terminal should work directly; if it's an old terminal that was already open before this change, the PATH change does not apply to it, so just open a new one.
- Transition notes support page_turn / wipe / match_cut / push_dissolve / custom; by default these are done in FFmpeg post; only invoke a generative transition when there is genuine narrative value. Details in `spec/04-shot-router.md`.
- **Post-assembly grading is done in `xfadeConcat` in `lib/film-assemble.ts`** (shared by `/api/assemble` and `scripts/render-film.mts`): before concatenation, apply the same `GRADE` to every shot (muted blue-gray shadows + warm sepia midtones, brightening and lifting the black point to counter i2v's tendency to be dark, `saturation 0.85`, protecting highlights) so separately-generated shots are unified; at the end `fadeOutDur` (default 1.0s) fades to black. `GRADE` is a single tunable constant; fine-tune darker/grayer here. i2v generation still uses `generate_audio:false` (video is always silent); the audio track is added in the **separate audio pass below**, keeping it separable and re-mixable.
- **Audio pass = the last step before export (landed 2026-07-19)**: `scripts/add-audio.mts` adds to the silent film (1) diegetic foley matched to the visuals + (2) one restrained score, then FFmpeg-mixes and muxes into `final/<slug>.mp4`. Everything goes through fal: foley uses **MMAudio V2** (`fal-ai/mmaudio-v2`, $0.001/s, video→audio, takes `video_url`+`prompt`+`negative_prompt`+`duration`, returns a video with audio), score uses **Lyria 2** (`fal-ai/lyria2`, $0.10/30s, text→music, no duration parameter → loop/trim in post), both already price-checked via fal MCP (see `lib/models.ts` `AudioModel`/`MMAUDIO_V2`/`LYRIA2`). Sound is **declarative**: cues are written in each beat's `sound{diegetic[],ambient}` in `data/presets/<slug>.json` (the RI five acts are filled in: writing scratches/page turns/stamping, footsteps/door closing, water pouring/bench/pen tip, river waves/printing press/the rise and fall of a speaker's voice/crowd murmur, keyboard/children playing), compiled by `compileSoundPrompt`/`compileWholeFilmSoundPrompt`/`compileMusicPrompt` in `lib/prompt-compiler.ts` (**no hand-written prompts**, reusing the STYLE mood); `SOUND_NEGATIVE` excludes score/clear human voices/subtitles/modern noise from the foley — crowds and speeches are only **indistinct, unintelligible ambient sound**, holding the "no dialogue/subtitles" red line; `MUSIC_NEGATIVE` excludes horror/drum beats/vocals, etc.
  - Two source modes are auto-detected: if `renders/<slug>/clip_XX.mp4` exists (a real render-film run happened) → **per-clip**, generate foley shot by shot and place it at that shot's start point on the film timeline per `clipOffsets()` (the same offset formula as `xfadeConcat`); if only `final/<slug>.mp4` exists (like the current 12.67s demo) → **whole-cut**, run one merged prompt over the whole film. Mixing is in `muxFilmAudio()` in `lib/film-assemble.ts`: each foley track is `adelay`ed to its start point, the score is filled via `-stream_loop` then ducked (`musicGainDb` default -13dB), all tracks are forced to stereo/48k (MMAudio output is mono, and without forcing it the score would collapse to mono), the ending `afade`s out with the visuals, `alimiter` prevents clipping, and the video is `-c:v copy` (not re-encoded).
  - Idempotent & cost-saving: on the first run the silent film is saved to `renders/<slug>/silent-master.mp4` as a durable silent source, and every subsequent mux overwrites `final`; `--remix` (reads `data/scenes/generated/<slug>/audio-provenance.json` to reuse already-generated audio) only re-mixes and **no longer calls fal**, used for adjusting gain. Paid runs require `--yes`, and >$5 requires `--confirm`. Provenance records request_id/model/prompt/parameters/cost, and no secrets. Run `node scripts/sync-public.mjs` when done. The current demo already has audio: whole-cut, foley $0.013 + music $0.10 = **$0.113**, and both `final` and `public/films` are h264+aac stereo.
- **Real-time transparency on the Film page**: during a real i2v run, `components/film/FilmView.tsx` shows the status shot by shot, the motion prompt actually compiled for that shot, and an inline preview as soon as each shot is produced (not waiting for the whole batch); the simulated/demo environment still uses the old linear checklist.
- **Implementation surface (2026-07-19 pivot landed)**: storyboard frames = `app/api/storyboard/frame` (generate/edit_add_asset/edit_prompt), T2I see `T2I_MODELS` in `lib/models.ts` (`nano-banana-2` default / `flux-2-pro` / `seedream-4` / `imagen-4`); shots = `app/api/shot/generate`+`app/api/shot/status` (queue), I2V see `I2V_MODELS` (`kling-v3-turbo-std` default / `happy-horse` / `veo3.1-hero`, `generate_audio:false`); the whole film = local `scripts/render-film.mts` reads `data/scenes/generated/<slug>/film-manifest.json` (written by `/api/generate/start`) → shot-by-shot I2V → FFmpeg xfade → `final/<slug>.mp4`, **paid runs require `--yes`, and hero/over-$5 requires `--confirm`**, `placeholder` frames are skipped.

- **Generative main-path endpoint verification (2026-07-19; prices change at any time, and `verifyBeforeCall` endpoints still need an MCP recheck before the first paid call)**: `fal-ai/nano-banana-2/edit` (Gemini 3.1 Flash Image), `fal-ai/flux-2-pro`, `fal-ai/kling-video/v3/turbo/standard/image-to-video` ($0.112/s), `alibaba/happy-horse/image-to-video` ($0.14/s@720p), `fal-ai/veo3.1/image-to-video` ($0.20/s@720p, no audio, hero). Collage rules for the Direction Two five acts: at most 7 archive / illustration references per frame, and the same source asset used in at most 2 scenes; the previous frame is no longer chained as a cross-film style anchor, with a unified style maintained by the shared prompt.
- **Style prompt color grading (2026-07-19)**: the `STYLE.palette`/`STYLE.mood`/`AVOID` blocks in `lib/prompt-compiler.ts` were given feedback that they were "too scary" rather than low-saturation retro. The palette's primary color has been changed from cool blue-gray to warm sepia + faded cream (blue-gray demoted to a weak accent, explicitly requiring soft, even lighting, neither dark nor high-contrast), the mood dropped "slightly haunting" and became warm-toned nostalgia (calm, not eerie/somber/haunting), and `AVOID` starts by hard-excluding horror/eerie/creepy/ominous/ghostly/deep shadows/low-key lighting/heavy vignette/cool gray/sickly green-cyan/high contrast. The three blocks are reused by T2I/I2V/edit, so one change unifies the whole film.
- **T2I aspect-ratio parameter format (2026-07-19 gotcha)**: the 16:9 parameter format differs across model families — the Gemini/Imagen family takes the string `aspect_ratio:"16:9"`, while the FLUX/Seedream family's `image_size` only takes `{width,height}` (or the `landscape_16_9` enum), and feeding the `"16:9"` string gets rejected outright by fal → the route silently falls back to a labeled placeholder (this was the root cause of flux always producing a placeholder). It is now declared per-model by `ImageModel.aspectRatioValue`, and the route no longer hardcodes it. The new candidates `seedream-4` (`fal-ai/bytedance/seedream/v4/text-to-image`) and `imagen-4` (`fal-ai/imagen4/preview`) are both prompt-only (no reference contact sheet fed in); their endpoints/prices/parameters are still `verifyBeforeCall:true` and need an fal MCP recheck before the first paid call.


# Into Place UI Redesign Rules
- Preserve all routes, data flows, APIs, scripts, and generation workflows.
- Do not delete or rewrite functional components for visual reasons.
- Work only on the redesign branch and inspect before editing.
- State planned files, implement one phase, run build, then stop.
- Style: tactile archival collage, editorial, spatial, refined, readable.
- Use restrained colors, clear typography, purposeful texture, and selective motion.
- Avoid SaaS patterns, gradients, glassmorphism, pill UI, random scrapbook decoration, and tourism aesthetics.
- Keep changes responsive, accessible, easy to review, and easy to revert.

## Redesign log
- **Type system moved to three families (developer directive 2026-07-19, supersedes the two-family scheme below)** — **Freckle Face** for display/page titles, **Chelsea Market** for body/reading copy, **Inria Sans** for microcopy / UI labels (bold for buttons). All three from `next/font/google` in `app/layout.tsx` (`--font-freckle|chelsea|inria`). Only `--font-display` changed → `var(--font-freckle)`; `--font-body`/`--font-hand` stay Chelsea, `--font-ui`/`--font-sans`/`--font-typewriter` stay Inria. Because titles resolve through `--font-display` (h1/h2/h3/.font-display in `globals.css`), the swap is global with no per-file edits. `tsc --noEmit` clean; `npm run build` NOT run (dev live).
  - **Gotcha (must follow): the `variable` class from `next/font` must be attached to `<html>`, not `<body>`.** The `--font-display/ui/body` generated by `@theme` are defined on `:root` (= `<html>`), and their values are `var(--font-freckle/chelsea/inria)`. CSS variables are evaluated on **the element that declares them** — if the next/font variables are only attached to `<body>`, then when `<html>` computes `--font-ui` the inner `var(--font-inria)` is undefined → the whole variable becomes IACVT (invalid at computed-value time) and empty → body inherits empty → `font-family: var(--font-ui)` fails → the whole site falls back to Tailwind's default font stack (appearing as "the font didn't take / looks like Inter", very deceptive, since it looks normal at all three layers of HTML/CSS/woff2). Fix: in `app/layout.tsx`, move the three `.variable` classes from `<body>` to `<html className={...}>` (Next's official approach). This trap was already lurking in the previous two-font version, it just wasn't traced to this layer. The decisive diagnostic step: in an incognito window console, run `getComputedStyle(document.body).getPropertyValue('--font-ui')` — an empty value means you're hit.
  - **Freckle Face narrowed to only `h1` (2026-07-19 feedback: the outlined typeface is too heavy on small headings).** In `globals.css`, `h1 → var(--font-display)` (Freckle) + `letter-spacing:0.045em` (the developer asked for wider letter-spacing on large titles); `h2, h3, .font-display → var(--font-body)` (Chelsea). Reason: the semantics of the `.font-display` class have drifted — it is used heavily on `text-sm/base/xs` small-text headings (Library/Story/Storyboard/Archive and 16 other files), and making it = Freckle would turn all the small text heavy, so it falls back to Chelsea. The only true large title is the `<h1>` in `app/page.tsx`; the `font-display` class has been removed from that h1 (otherwise class specificity would beat the `h1` element selector and pull it back to Chelsea), as has `tracking-tight` (negative letter-spacing, which would override the h1's letter-spacing). For new page-level large titles, use the semantic `<h1>` to get Freckle + wide letter-spacing automatically; do not use `.font-display`.
